# Contributing to Retro-Os

Thank you for your interest in contributing to Retro-Os! To maintain the security and stability of the repository, we enforce a strict contribution workflow.

## Strict Rules
1. **No Direct Pushes**: You cannot push directly to the `master` branch.
2. **Pull Requests Only**: All changes must be submitted via a Pull Request (PR).
3. **Approval Required**: Every PR must be reviewed and approved by the repository owner (@Mr-Charvaka) before merging.
4. **Merge Requirement**: Code can only be added to the repository if and only if the merge is completed after review.

## Contribution Workflow
1. **Fork** the repository (or create a new branch if you have permissions).
2. **Implement** your changes.
3. **Test** your changes thoroughly.
4. **Create a Pull Request** against the `master` branch.
5. **Wait for Approval**: The repository owner will review your code.
6. **Merge**: Once approved, the code will be merged into the main codebase.

## Security
If you find a security vulnerability, please do NOT open an issue. Instead, contact the repository owners directly.
